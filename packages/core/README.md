# core_logic

Pure-Rust library shared by the node and all clients. Holds the parsing
pipeline, ASN.1 decoders, BLE protocol, and the analyzers (ego-MAC,
vehicle tracking, tracking warnings). No hardware deps, no async runtime.

## Modules

- `pcap_parser` Radiotap then 802.11 MAC then LLC/SNAP then GeoNetworking
  then BTP-B then ITS payload.
- `parser/` decoders for CAM v1 (EN 302 637-2) and CAM v2 (TS 103 900),
  plus DENM v1 (EN 302 637-3). The router dispatches by BTP port.
- `c_its` and `asn1` Rust facade over the C ASN.1 sources generated by
  asn1c. The C code is compiled via `cc` and bound with `bindgen` in
  `build.rs`.
- `ble_protocol` 4-byte fragment header used to chop large payloads
  into BLE notifications, plus the `BleReassembler` used on the client.
- `ble_handshake` 6-byte session handshake that negotiates protocol
  version and the max chunk size used for the rest of the session.
- `ego_mac` rolling RSSI window with a stability score that picks the
  MAC most likely to belong to the host vehicle.
- `vehicle_tracker` links pseudonym changes into stable virtual IDs
  using kinematic continuity (position, speed, heading, standstill).
- `tracking_warning` fires per-vehicle warnings when a foreign car has
  been visible for too long or has been followed for too far.
- `config` shared config structs that travel between client and node:
  `RepeatModeConfig`, `InjectionConfig`, `TrackingWarningConfig`,
  `NodeConfig`, `InjectionStatus`.
- `crc32` CRC used for the injection-loopback filter.

## ASN.1 sources

ETSI schemas live under `resources/asn1/<module>/` and are regenerated
with:

```
chmod +x packages/core/resources/asn1/build_asn1_c.sh
packages/core/resources/asn1/build_asn1_c.sh
```

The resulting C files are checked in so a normal `cargo build` does not
need asn1c.

## Tests

`cargo test --package core_logic`

Integration tests live under `tests/`; the largest one parses a real
ITS-G5 capture and diffs the output against a reference jsonl.
